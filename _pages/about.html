Overview

Hi, I'm Dom. I'm a software engineer based in the United Kingdom.

Here you can read about me, my background, and what I'm currently working on. This site is also home to articles I write for my blog.

For a formal take on my background to date, have a look at my CV.

Professional Background

I'm a software engineer at Applied Systems, who are the market leaders in cloud based insurance management software. For a jargon-free example of what that means, I'm currently leading development of a set of .NET 6.0 microservices that calculate insurance premiums in real time according to pricing models specified by insurers.

At Applied I work with C#, .NET 6.0 and SQL Server on the backend, and sometimes work on web applications using ASP.NET Core, JavaScript and HTML/CSS. I've also made use of Golang and PowerShell to write CLI tools.

Prior to Applied Systems I worked as a statistical analyst at an insurance company called First Central. My role there aligned closer with that of a data or research scientist, where freedom was given for teams to work in a research and development driven environment. The end goal was insurance pricing optimisation using machine learning techniques. Think principal component analysis, clustering, gradient boosting and other statistical methods, used to extract signal from streams of noisy data. Tech used was mainly the Python data science stack (NumPy, SciPy, Matplotlib, Pandas, etc), SAS and SQL.

Personal Background

Outside of work I enjoy learning about and dabbling in different technologies. This site as an example is backed by Jekyll, the Liquid template language and SCSS. Other personal projects involved use of C++, Python, and TypeScript.

Light++: my personal light theme for VSCode. Find it by searching for Light++ under Extensions.

I'm currently planning to revisit development of a chess engine I started working on in 2020. It's written in C++ and exposes a CLI compatible with the Universal Chess Interface, meaning that once ready it will be able to compete with other open source engines on the Computer Chess Rating List.

I'm an advocate for open source, and incorporate the UNIX philosophy in my approach to software development. From system design down to optimising my local development environment, I'm driven by modularity, testability, and reduction of complexity. I favour best practices, clean code and clean architecture.

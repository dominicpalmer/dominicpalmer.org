Professional Background

I'm currently based in Brighton, but I've spent time living in London too. I'm a software engineer working remotely for the Belfast office of Applied Systems, who are the market leaders for cloud based insurance management software. For a jargon-free example of what that actually means, the main project I work on is a set of .NET Core microservices that calculate insurance premiums in real time according to pricing models specified by insurers and insurance brokers.

The tech stack I'm using at Applied is predominantly C#, .NET Core and SQL Server, although I also work on internal web tooling using ASP.NET, JavaScript and HTML/CSS. While I mainly work on backend services, I've also built an ASP.NET Core web portal for managing insurance document templates, a lightweight internal CLI tool in Golang to probe production databases, and contributed toward various Windows desktop GUI applications. So the day to day can vary.

Prior to Applied Systems I worked as a statistical analyst at a UK-based motor insurance company called First Central. My role there aligned closer with that of a data or research scientist, where freedom was given for teams to work in a research and development driven environment. The end goal was insurance pricing optimisation using machine learning techniques. Think along the lines of principal component analysis, clustering, gradient boosting and other statistical methods, all used to extract signal from streams of noisy data. Tech used was mainly the Python data science stack (NumPy, SciPy, Matplotlib, Pandas, etc.), SAS and SQL.

I'm currently planning to revisit development of a chess engine I started working on in 2020. It's written in C++ and exposes a CLI compatible with the Universal Chess Interface, meaning that once ready it will be able to compete with other open source engines on the Computer Chess Rating List.
